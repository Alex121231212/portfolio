# Aida

Платный SaaS-сервис генерации академических работ (дипломы, курсовые, рефераты, статьи, отчёты по практике) с двумя режимами генерации, автопоиском научных источников, биллингом по токенам и тарифной системой.

## Два режима генерации

### 2.0 — Агентский (флагман)

Полноценный AI-агент, который сам ищет источники, выбирает структуру и итеративно дорабатывает работу через tool_use. Ничего не нужно загружать вручную.

```
LLM получает ТЗ + tools
   ↓
while not done:
    Claude решает что делать → вызывает tool
    Backend выполняет tool → возвращает результат
    Claude анализирует → следующий шаг
   ↓
Готовая работа в DOCX
```

Доступные tools агента:

| Tool | Что делает |
|------|------------|
| `search_sources(query)` | Поиск по 20+ научным базам, дедупликация, ранжирование, импорт PDF |
| `read_source_chunks(query, max)` | Поиск релевантных фрагментов в загруженных источниках (BM25) |
| `get_source_summary(source_id)` | Краткое содержание конкретного источника |
| `get_structure()` | Текущая структура работы со статусами разделов |
| `update_structure(action, ...)` | Изменение структуры: add / remove / rename / reorder |
| `write_section(section_id)` | Генерация конкретного раздела с иерархическим контекстом |

### 1.0 — Линейный (ручной)

Классический режим для пользователей, которые хотят сами контролировать процесс. Линейный 4-шаговый пайплайн:

1. Анализ ТЗ
2. Анализ источников (загружаются пользователем)
3. Анализ примера (для подражания стилю)
4. Потоковая генерация через SSE

Этот режим был исходным; режим 2.0 построен поверх него и переиспользует часть инфраструктуры (биллинг, генерация, экспорт).

## Воронка автопоиска источников (режим 2.0)

Ключевое инженерное решение режима 2.0 — поиск научных статей построен как воронка из 5 слоёв, где **LLM подключается только в конце**, когда данных мало:

```
Слой 1: Поиск                        0 токенов
    paper-search-mcp → 20+ научных баз параллельно
    (arXiv, PubMed, OpenAlex, Semantic Scholar, Crossref,
     dblp, bioRxiv, CORE, Europe PMC, ...)
    ↓ ~100 результатов

Слой 2: Дедупликация                 0 токенов
    DOI / нормализованный заголовок + мердж метаданных
    (у одной базы PDF, у другой — citations)
    ↓ ~40 уникальных

Слой 3: Ранжирование                 0 токенов
    BM25 по title+abstract
    + цитируемость / возраст / наличие PDF
    ↓ Top 10–15

Слой 4: LLM-фильтр                   ~$0.001
    Дешёвая модель оценивает релевантность абстрактов (1–10)
    ↓ Top 5–8 (score ≥ 7)

Слой 5: Импорт                       ~$0.01
    Скачивание PDF (Open Access)
    Чанкинг (400 токенов, 80 overlap)
    Суммаризация (дешёвая модель)
    → BM25-индекс в проекте Aida
```

**Итого ~$0.01 за полноценный научный ресёрч по 20+ базам**, против $1+ при наивном подходе «загрузи всё в LLM». Никаких аккаунтов Google Scholar / NotebookLM, никакой капчи, никакого риска бана.

## Стек

- **Backend:** Python, Anthropic SDK, многошаговый агент с tool_use
- **БД:** MongoDB (с индексами на users / projects / token_usage / payments)
- **Кэш / очереди:** Redis (заглушка под Celery)
- **Frontend:** React
- **Reverse proxy:** Caddy с автоматическим Let's Encrypt
- **LLM:** Claude Sonnet (генерация и агент) + Grok-fast (суммаризация, фильтрация в воронке)
- **Поиск научных статей:** [paper-search-mcp](https://github.com/openags/paper-search-mcp) (arXiv, PubMed, OpenAlex, Semantic Scholar, Crossref, dblp, bioRxiv, CORE, Europe PMC, ...)
- **Развёртывание:** Docker Compose, GitHub Actions CI/CD (lint → build → deploy on VPS via SSH)
- **Мониторинг:** healthcheck-эндпоинт, log'и операций; stub'ы под Sentry и Prometheus

## Ключевые решения

### Двухмодельная экономика

Тяжёлые задачи (агент, генерация связного текста) — Claude Sonnet. Лёгкие (суммаризация чанков для иерархического контекста, LLM-фильтр в воронке поиска) — Grok-fast. Это держит стоимость работы вменяемой при качественной генерации.

### Иерархический контекст генерации

Чтобы главы получались связными, но не падали из-за переполнения контекста, каждая глава видит:
- свой полный текст
- summary всех остальных глав

Summary'и считаются дешёвой моделью один раз, переиспользуются для каждой главы.

### Streaming через SSE

Генерация финального текста идёт через Server-Sent Events с раздельными событиями `step` / `content`. Пользователь видит как пишется работа в реальном времени.

### Биллинг как first-class

Каждый LLM-вызов записывается в `token_usage` (input/output токены, стоимость, проект). Перед генерацией — проверка лимита (HTTP 402 если исчерпан). Тарифы: Free / Basic / Pro / Enterprise с разными лимитами и максимальным числом проектов. Интеграции с ЮKassa и Stripe реализованы.

### Production-инфра с первого дня

HTTPS через Caddy + Let's Encrypt, MongoDB-индексы под фактические запросы, бэкап-скрипт с ротацией 7 дней, rate-limiting, brute-force protection, healthcheck для мониторинга, GitHub Actions для деплоя через SSH.
