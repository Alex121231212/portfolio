# Aida

AI-генератор академических работ (дипломы, курсовые, рефераты, статьи, отчёты по практике) с автоматическим поиском научных источников и агентской системой генерации.

## Что делает

- Принимает ТЗ от пользователя (тема, тип работы, объём, дисциплина)
- **Автоматически ищет научные источники** по 20+ публичным научным базам
- Сам выбирает релевантные статьи, скачивает PDF, чанкует, индексирует
- Генерирует работу по разделам через агентскую систему с самопроверкой
- Экспортирует DOCX по ГОСТ 7.32 / 7.0.5

## Архитектура

```
ТЗ пользователя
   ↓
[Слой 1] Поиск по 20+ научным базам параллельно (paper-search-mcp)
   ↓ ~100 результатов, 0 токенов LLM
[Слой 2] Дедупликация по DOI / нормализованному заголовку
   ↓ ~40 уникальных, 0 токенов
[Слой 3] BM25-ранжирование по title+abstract + цитируемость + год
   ↓ Top 10–15, 0 токенов
[Слой 4] LLM-фильтр: оценка релевантности по абстрактам
   ↓ Top 5–8, ~$0.001
[Слой 5] Скачивание PDF, чанкинг, суммаризация → BM25-индекс
   ↓ ~$0.01

Агентская генерация:
   while not done:
       LLM решает что делать → вызывает tool
       (search_sources, read_chunks, update_structure, write_section, ...)
       Backend выполняет → возвращает результат
       LLM анализирует → следующий шаг
   → готовая работа в DOCX
```

## Стек

- **Backend:** Node.js, Python (для научного поиска), Docker
- **LLM:** Claude Sonnet (генерация), Grok Fast (суммаризация и фильтрация) через прокси-шлюз
- **Поиск:** paper-search-mcp (arXiv, PubMed, OpenAlex, Semantic Scholar, Crossref, dblp, bioRxiv, CORE, Europe PMC...)
- **Frontend:** React + Caddy
- **Экспорт:** DOCX по ГОСТ
- **Биллинг:** кредитная система, реферальная программа (5 уровней партнёрки)

## Ключевые решения

- **«LLM подключается только когда мало данных»** — первые 3 слоя поиска работают без токенов вообще (HTTP + строки + BM25). LLM фильтрует только финальные 15 кандидатов. Итого ~$0.01 за полноценный научный ресёрч вместо $1+ при наивном подходе.
- **Открытые научные API вместо парсинга Google Scholar / NotebookLM** — нет аккаунтов, нет капчи, нет риска бана, и API публичные.
- **Агент с tool_use вместо линейного пайплайна** — LLM сама решает когда искать источники, когда менять структуру, когда переписывать раздел. Цикл self-improvement до приемлемого качества.
- **Дедупликация и мердж метаданных** — у одной статьи в одной базе есть PDF, в другой — цитаты. Мержим по DOI, получаем максимально полную карточку.
