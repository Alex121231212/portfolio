# AstroTrainer

Геймифицированный тренажёр жизни через астрологию. Пользователь общается с AI-персонажами планет своей натальной карты — у каждой планеты свой архетип, тон и стиль реакции.

## Что делает

- Регистрация по дате/времени/месту рождения → расчёт натальной карты
- Генерация AI-персонажей планет с параметризацией от архетипа, знака и дома
- Диалог пользователя с персонажем
- В roadmap (MVP-2/3): игровые сценарии, прогресс, аналитика взаимодействий

## Ключевые решения

### Astro-расчёты в отдельном Python-микросервисе

pyswisseph (Swiss Ephemeris) не имеет JS-эквивалента нужного качества и точности. Изоляция на FastAPI с REST-интерфейсом даёт чистое разделение: TypeScript-сервер не тащит C-зависимости, Python-сервис не знает про бизнес-логику и пользователей.

### Cookie-based auth без JWT

argon2id хеш паролей + httpOnly + sameSite cookies. Меньше движущихся частей чем у JWT, нет проблемы invalidation, нет вопросов «куда хранить» на фронте. CSRF закрывается sameSite + double-submit-token при необходимости.

### Биндинг сервисов на localhost

Наружу торчит только Next.js через nginx. Postgres и astro-сервис недоступны извне даже при ошибке firewall — defense in depth.

### Каждая планета как отдельный персонаж

System prompt параметризован архетипом + знаком + домом. Один и тот же код генерации даёт уникальные «голоса» персонажей; добавление новой планеты или модификация архетипа — изменение одной строки в конфиге, без правки кода.

### Корректная работа с часовыми поясами

Точная натальная карта требует местного времени рождения с учётом исторических смен часовых поясов. tz-lookup по координатам места рождения определяет IANA-зону, далее всё считается в UTC и приводится назад только при отображении.
