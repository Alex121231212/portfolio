# AstroTrainer

Геймифицированный тренажёр жизни через астрологию. Пользователь общается с AI-персонажами планет своей натальной карты — у каждой планеты свой характер, голос и стиль реакции.

## Что делает

- Регистрация по дате/времени/месту рождения → расчёт натальной карты
- Генерация AI-персонажей для каждой планеты (Солнце, Луна, Меркурий и т.д.)
- Чат с персонажем планеты в формате диалога
- Сценарии-задания: «спроси у Марса как действовать», «обсуди с Венерой отношения»
- Прогресс пользователя, статистика взаимодействий

## Стек

| Слой | Технология |
|---|---|
| Frontend + API | Next.js 14 (App Router), TypeScript, Tailwind |
| ORM | Prisma + PostgreSQL 16 |
| Astro-расчёты | Python 3.11, FastAPI, pyswisseph |
| LLM | Claude через прокси-шлюз |
| Auth | argon2id + httpOnly cookies (без JWT) |
| Hosting | self-hosted Docker, nginx-proxy |

| Сервис | Порт | Биндинг |
|---|---|---|
| Next.js | 3100 | публичный через nginx |
| FastAPI astro | 8100 | localhost only |
| Postgres | 5435 | localhost only |

## Ключевые решения

- **Astro-расчёты вынесены в отдельный Python-микросервис** — pyswisseph не имеет JS-аналога нужного качества, поэтому крутой профит от изоляции на FastAPI и общения через REST.
- **Cookie-based auth без JWT** — argon2id хеш паролей + httpOnly + sameSite=lax. Меньше движущихся частей, нет проблемы invalidation.
- **Биндинг сервисов на localhost** — наружу торчит только Next.js через nginx, Postgres и astro-сервис не доступны извне даже при ошибке firewall.
- **Каждая планета как отдельный персонаж** — system prompt параметризован архетипом + знаком + домом, что даёт уникальные «голоса» при общем коде.
