# Kontent Zavod

Универсальная автоматизированная система производства контента: по заданной нише система сама анализирует поисковый спрос, формирует очередь тем, генерирует и публикует контент. Смена ниши делается **только через настройки**, без изменения логики.

## Что делает

- **Анализ спроса** — парсинг поисковых подсказок и формирование очереди тем
- **Классификация интентов** — коммерческий / экспертный / риск-страх / upsell / кейсы
- **Генерация контент-юнитов** — статьи (4–7к знаков с H1–H3, meta), видео (TBD), посты в соцсети
- **SEO + GEO-оптимизация** — структура под классическую выдачу и под ответы AI-ассистентов (Алиса, GigaChat, zero-click)
- **Публикация** — WordPress по API, Telegram, VK, Яндекс Дзен
- **Защита от самоканнибализации контента** — проверка уникальности на смысловом уровне, а не по тексту

## Стек

- **Backend:** Python, REST API, модульная архитектура
- **БД:** PostgreSQL
- **Развёртывание:** Docker Compose
- **LLM:** через прокси-шлюз
- **Источники тем:** API поисковых подсказок
- **Storage:** MinIO (для видео, артефактов)

## Поддерживаемые ниши

Система работает с любой нишей, где есть поисковый спрос и экспертный/коммерческий контент: сертификация продукции, торги по банкротству, юр. услуги, e-commerce, медицина, образование, БАДы и т.д.

## Ключевые решения

### Универсальные сущности вместо специфичных

В коде нет «статьи» — есть `content_unit`, который **может иметь несколько форматов**. Один контент-юнит → текст для блога, скрипт для видео, пост для соцсети. Это позволяет менять нишу одним переключением конфига.

### Ниша = конфигурация

Каждый проект задаётся набором параметров: ключевые слова, коммерческие/информационные триггеры, тон, юридические ограничения, лимиты, регион, источники, LLM-провайдер. Никакой логики в коде, специфичной для ниши.

### Многоуровневая проверка уникальности

Чтобы при долгой работе в одной нише избежать повторов:

1. **Уникальность темы** — проверка на смысловом уровне (не по тексту), перед добавлением в очередь
2. **Уникальность интента** — одна формулировка, разные интенты → допустимо
3. **Уникальность контент-юнита** — не было ли уже создано для этой темы + интента + формата

При обнаружении повторов система не молча дропает, а решает: отклонить / понизить приоритет / пометить как производную.

### LLM-экономика прозрачна

Каждый LLM-вызов логируется: модель, токены in/out, стоимость, версия промпта, провайдер. Можно подбирать оптимум по цене на разных этапах пайплайна.

### Идемпотентность пайплайна

Сущность `pipeline_run` со статусами этапов. При перезапуске не происходит повторной генерации того, что уже сделано. Очередь задач + ретраи по сбоям API. Флаги включения этапов: `collect / intent / seo / generate / geo / publish` — можно отключать модули без правки кода.

### GEO/AEO как first-class

Контент изначально структурируется под извлечение AI-ассистентами:

- чёткие однозначные ответы на вопросы
- краткие смысловые блоки и определения
- информация извлекается без интерпретации
- никакой «воды» во вводных абзацах

Это даёт zero-click видимость в Алисе/GigaChat помимо классической SEO-выдачи.

### Полу-автомат для MVP

Первая итерация — полу-автоматическая публикация (одобрение в админке перед публикацией). Полная автоматизация включается после стабилизации качества генерации.
