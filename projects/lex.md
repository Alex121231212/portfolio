# Lex

AI-SaaS для автоматизации юридической рутины: интейк клиентов, поиск судебной практики, генерация процессуальных документов, подача через ГАС Правосудие.

## Что делает

- **Интейк** — запись разговора с клиентом, транскрипция, квалификация дела
- **RAG по судебной практике** — гибридный поиск по парсингу sudact.ru
- **Генерация процессуальных документов** — иски, отзывы, ходатайства; редактирование в TipTap, версионирование
- **Подача документов** — браузерный агент через ГАС Правосудие со streaming-логом для юриста
- **Мониторинг хода дел** — оркестратор фоновых задач (в активной разработке)

## Ключевые решения

### Multi-tenancy

`tenantId` фильтр на всех запросах + дополнительная проверка владения объектами при join'ах (defense in depth). Кейсы / документы / клиенты не утекают между tenant'ами даже при ошибочной выборке.

### Гибридный поиск судебной практики

`pgvector` даёт семантическую близость к запросу, `pg_bm25` (Paradedb) — точные совпадения терминов и норм. Результаты сливаются с весом, что важно для юр-домена: цитата нормы должна находиться даже если запрос сформулирован совсем иначе.

### Streaming через SSE на всех долгих операциях

Транскрипция, генерация документа, подача через ГАС — всё стримится через SSE с раздельными событиями (`status` / `chunk` / `done`). При replay завершённой задачи бэкенд отдаёт сохранённое финальное состояние, а не открывает новый стрим.

### Browser-agent с persistable state

Сессия подачи (filing) живёт в БД, не только в памяти. При рестарте API non-terminal сессии помечаются `cancelled` и юрист получает понятный статус вместо вечного «запускаю браузер».

### Tool-using агент с idempotency

Генерация документа защищена от двойных кликов: повторный запрос на `(caseId, type)` с пустым черновиком младше 60 секунд блокируется, чтобы не создавать параллельные LLM-стримы и двойной биллинг.

### Sprint-driven roadmap

Шесть спринтов: скелет → интейк → RAG → docgen → browser-agent → monitor. Готов и в активной эксплуатации функционал интейка, RAG, генерации документов и подачи; мониторинг дел в работе.
